
# CONTEXT.md
## Проект: Планка (рабочее название)
Цифровой коммуникационный инструмент для Андрюши

---

# 1. Миссия проекта

Создать специализированное устройство для неговорящего подростка с РАС,
которое обеспечивает устойчивый, предсказуемый и автономный способ коммуникации.

Это не "приложение на планшете".
Это отдельный инструмент общения.

Устройство должно:
- Всегда быть готовым к печати
- Не требовать запуска
- Не позволять случайного выхода
- Быстро и стабильно отображать текст
- При необходимости проговаривать его вслух
- Быть рассчитанным на долгосрочное использование

---

# 2. Пользователь

Имя: Андрюша  
Возраст: 12 лет  
Особенности:
- Неговорящий
- Знает буквы
- Запоминает визуальное написание слов, которые имеют для него значение
- Обладает хорошей памятью и высокой наблюдательностью

## Преемственность

Проект продолжает уже существующую коммуникационную практику:

1. Бумажные квадратные листочки с ручками (расположены дома).
2. Магнитный «Эрудит» — регулярное составление слов и фраз.
3. Визуальная память слов.

Мы не создаём новый способ общения.
Мы переводим уже освоенный способ в устойчивую цифровую форму.

---

# 3. Ключевые принципы

1. Предсказуемость.
2. Отсутствие скрытых состояний.
3. Нулевая случайность.
4. Минимум жестов.
5. Отсутствие автоисправления по умолчанию.
6. Стабильная раскладка клавиатуры.
7. Устройство не должно ощущаться как обычный планшет.

---

# 4. Основной сценарий

1. Устройство включается.
2. Автоматически открывается приложение.
3. Экран разделён горизонтально:
   - Верхняя половина — текст.
   - Нижняя половина — клавиатура.
4. Андрюша печатает.
5. Текст можно:
   - Показать.
   - Проговорить.

---

# 5. Функциональные требования (MVP)

## 5.1 Основной экран

- Ландшафтная ориентация (заблокирована).
- Верхняя зона:
  - Крупный масштабируемый текст.
  - Возможность отображения текста во весь экран.
- Нижняя зона:
  - Кастомная клавиатура RU/EN.
  - Пробел.
  - Backspace (удержание = ускоренное удаление).
  - Очистка с защитой от случайного нажатия.
  - Переключение языка через удержание.

## 5.2 Проговаривание (TTS)

- Кнопка «Сказать».
- Проговаривание текста без автоисправлений.
- Возможность услышать собственный текст.
- Настройки голоса (родительский режим).
- В перспективе — кастомизация голоса.

## 5.3 Переворот текста

При наклоне устройства «от себя»:
- Текст в верхней части переворачивается на 180°.
- Клавиатура остаётся в стандартной ориентации.
- Используются данные гироскопа/акселерометра.

## 5.4 Киоск-режим

Требования:
- Автозапуск при включении.
- Блокировка выхода.
- Блокировка системных жестов.
- Блокировка панели уведомлений.
- Невозможность запуска других приложений.

Родительский вход:
- Секретная комбинация клавиш.
- Либо PIN-код.

---

# 6. Надёжность

- Сохранение текста при краше.
- Восстановление состояния после перезагрузки.
- Локальное хранение.
- Возможность резервного копирования.

---

# 7. Аппаратная стратегия

## 7.1 Устройство не выбрано

Мы сознательно сохраняем свободу выбора hardware.

Проект ориентируется на свойства устройства, а не на конкретную модель.

## 7.2 Предпочтительный форм-фактор

Желаемое соотношение сторон: примерно 2:1 (в ландшафте).

Почему:
- Оптимальное деление 50/50 (текст/клавиатура).
- Крупные клавиши.
- Хорошая читаемость текста на расстоянии.
- Рациональное использование пространства.

## 7.3 Аппаратные требования

Экран:
- 10–12 дюймов
- IPS
- Высокая яркость
- Хорошие углы обзора
- 1920×1080 или выше
- Соотношение сторон близкое к 18:9 / 20:9

Датчики:
- Акселерометр
- Гироскоп

Производительность:
- Стабильность важнее мощности
- 4–8 ГБ RAM и выше (возможны улучшения)
- 64–128 ГБ storage и выше

Бюджет не является ограничивающим фактором.
Приоритет — комфорт, долговечность и надёжность.

# 8. Архитектурные варианты

## Вариант A — Нативный Android
+ Максимальный контроль kiosk
+ Стабильность
− Отдельный стек для iOS

## Вариант B — WebView + нативный shell
+ Быстрая разработка
+ Возможность кросс-платформы
− Возможные сложности с kiosk

Окончательное решение принимается после прототипирования клавиатуры.

---

# 9. Критерии успеха

- Андрюша регулярно использует устройство самостоятельно.
- Минимум фрустрации.
- Невозможность случайного выхода.
- Устройство воспринимается как отдельный инструмент коммуникации.

---

# 10. Долгосрочное развитие

- Логи и экспорт сообщений.
- Обновляемая клавиатура из репозитория.
- Light/Dark режим.
- Ограниченный шотбар фраз.
- Распознавание рукописного ввода.
- Потенциальная адаптация в store-продукт для других семей.
