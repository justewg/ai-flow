# PL-096 — Integration 2 / policy-first gating

Цель:
- начать читать verdicts из `runtime-v2` перед expensive path legacy runtime;
- оставить legacy shell runtime transport/effect layer, а policy читать уже из `v2`-контура;
- не делать full cutover.

Планируемый scope:
- rollout gate и budget gate для daemon/executor через `runtime-v2`
- первые stale/budget reads для supervisor path
- rollout только через `dry_run -> shadow -> single_task -> limited`

Definition of done:
- expensive path в legacy runtime не стартует без явного `runtime-v2` policy allow
- budget stop и rollout deny читаются из `runtime-v2`, а не из legacy ad-hoc логики
- интеграция остаётся reversible и не заменяет ещё весь control plane
