# PL-097 — Integration 3 / event bridge into runtime-v2 orchestrator

Цель:
- начать отдавать selected lifecycle transitions в `runtime-v2 applyEvent()`;
- оставить legacy shell transport/effect layer, но перестать держать review/wait/execution transitions только в файлах shell-state;
- не делать ещё full orchestrator cutover.

Планируемый scope:
- `runtime_v2_apply_event` bridge для selected legacy transitions;
- bridge для:
  - `human.wait_requested`
  - `human.response_received`
  - `review.finalized`
  - `execution.started`
  - `execution.finished`
- best-effort wiring в legacy scripts без жёсткого отказа всего flow при падении bridge.

Definition of done:
- selected legacy lifecycle transitions materialize-ятся в `runtime-v2` через `applyEvent()`;
- `runtime-v2` больше не узнаёт review/wait/execution state только через shadow projection;
- end-to-end smoke подтверждает цепочку `wait -> response -> review` через shell wrapper.
