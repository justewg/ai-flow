# PL-099 — Integration 5 / runtime-v2 primary source for WAIT_HUMAN

Цель:
- перевести `WAIT_HUMAN` в primary-source режим из `runtime-v2`;
- перестать считать `daemon_waiting_*` source of truth для обычного human wait;
- оставить review-feedback path вне scope этой фазы.

Планируемый scope:
- `human.wait_requested` и `human.response_received` должны не только зеркалиться в `runtime-v2`, но и хранить waiting metadata для reconcile;
- `runtime_v2_primary_context` и `runtime_v2_reconcile_primary_context` должны начать восстанавливать legacy `daemon_waiting_*` из `runtime-v2`;
- `task_ask.sh` должен отправлять полноценный waiting payload и для immediate-post, и для pending-post path.

Definition of done:
- `runtime-v2` хранит waiting metadata, достаточную для восстановления `daemon_waiting_*`;
- reconcile-wrapper восстанавливает legacy waiting context из `runtime-v2`;
- local smoke показывает, что после `human.wait_requested` legacy waiting files материализуются из `runtime-v2` без ручной записи;
- review-feedback path остаётся legacy-first и не расширяет scope фазы.
