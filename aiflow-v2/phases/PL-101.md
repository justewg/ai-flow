# PL-101 — Integration 7 / rebuild watchdog into a true supervisor

Цель:
- превратить watchdog из recovery-loop в supervisor, который только detect/freeze/stop/alert;
- убрать из watchdog implicit daemon restart / daemon tick как "лечение по умолчанию";
- использовать watchdog как policy-enforcer для containment, а не как скрытый второй orchestrator.

Планируемый scope:
- action model watchdog переводится с `SOFT/MEDIUM/HARD recovery` на supervisor actions:
  - `FREEZE_SAFE`
  - `FREEZE_EMERGENCY`
  - `STOP_EXECUTOR_AND_FREEZE`
  - `ALERT_BRANCH_SYNC`
- при anomaly watchdog:
  - переводит runtime в `SAFE`/`EMERGENCY_STOP`;
  - при необходимости делает только `executor_reset`;
  - пишет incident trail;
  - шлёт alert;
  - не вызывает `daemon_tick` и не переустанавливает daemon.

Definition of done:
- watchdog больше не делает daemon recovery side effects по умолчанию;
- anomaly path переводит runtime в containment mode и фиксирует incident;
- local smoke показывает, что supervisor freeze semantics работают без restart loop.
