# PL-104 — Integration 10 / full dry-run and shadow validation path

Цель:
- собрать один явный validation harness для controlled rollout modes;
- подтвердить, что `dry_run` и `shadow` режут expensive/side-effect paths предсказуемо;
- одновременно доказать, что inspection/context path остаётся живым и наблюдаемым.

Планируемый scope:
- reusable validation module внутри `runtime-v2/src`;
- CLI и shell wrapper для локального прогона;
- запись validation report в state dir;
- docs/update trail под новый validation path.

Definition of done:
- один запуск validation harness даёт отчёт по `dry_run` и `shadow` scenarios;
- harness проверяет, что `shadow` сохраняет read-only inspection path;
- отчёт пишется в `<state-dir>/runtime_v2_validation_report.json`;
- есть unit-test и shell smoke.
