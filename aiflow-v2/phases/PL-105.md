# PL-105 — Integration 11 / controlled single-task local v2 loop

Цель:
- подтвердить, что `single_task` rollout mode способен пропустить ровно один allowlisted task;
- прогнать canonical happy-path lifecycle одной задачи через `runtime-v2`;
- получить replayable local loop report вместо ручной последовательности event-команд.

Планируемый scope:
- reusable single-task loop module;
- CLI и shell wrapper;
- отчёт в state dir;
- фиксация результата в docs/worklog.

Definition of done:
- allowlisted task проходит `single_task` gate;
- denied task получает `rollout_single_task_denied`;
- canonical loop заканчивается в `reviewing` с canonical PR;
- shell smoke пишет `runtime_v2_single_task_loop_report.json`.
