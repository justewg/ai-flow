# PL-110

Title: Preserve freshly claimed active context until executor start

Goal:
- stop orphaned claims where `daemon_active_*` is written on claim and then cleared by `runtime_v2_reconcile_primary_context` before `executor_tick` can start the executor.

Scope:
- keep legacy active context during a short claim grace window when `runtime-v2` still has no `executing` task state.
- do not synthesize fake `execution.started` during claim.
- leave review and waiting reconciliation semantics unchanged.

Acceptance:
- after `CLAIMED_TASK_ID`, the next daemon tick still sees `daemon_active_task.txt` long enough to reach `executor_tick`.
- `runtime_v2_reconcile_primary_context` clears stale active context only after the grace window expires.
- local smoke shows `daemon_active_task.txt` survives when the store contains only `phase=ready` plus a fresh `daemon_last_claim_epoch.txt`.
