# PL-111

Title: Harden executor prompt build and tolerant event payload parsing

Goal:
- stop executor from failing before useful work starts because of shell expansion inside prompt templates
- stop best-effort `runtime_v2_apply_event` bridge from crashing on payload strings with trailing junk

Scope:
- rebuild `executor_build_prompt.sh` without unquoted heredoc expansion
- make `runtime_v2_apply_event.js` parse payload JSON tolerantly and fall back safely for best-effort bridge calls
- keep rollout and task execution semantics unchanged

Acceptance:
- prompt builder preserves literal backticks such as `` `USER_REPLY` `` and `` `task_ask blocker` `` without shell errors
- `runtime_v2_apply_event.js` accepts a payload like `{"k":"v"}TRAILING_JUNK` and applies the leading JSON object instead of crashing
- local smokes cover both regressions
