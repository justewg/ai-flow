# PL-112 — Post-canary hardening after `PL-110`

Цель:
- убрать три системных хвоста, которые проявились в первом реальном executor run `PL-110`;
- не лечить их операционными workaround-ами в задаче, а поднять до upstream flow behaviour;
- сохранить current limited-rollout contour без смены backend storage.

Планируемый scope:
- materialize `/.flow/shared` submodule в task worktree до старта executor;
- добавить executor-side fallback check, чтобы stale worktree без toolkit падала управляемо и с incident trail;
- исправить upstream `task_finalize` wrapper call для `run_commit_push_for_branch(repo_root, branch, ...)`;
- нормализовать `payload-json` на shell boundary до вызова `runtime_v2_apply_event.js`, чтобы `review.finalized` не тащил trailing junk.

Definition of done:
- fresh task worktree сразу содержит materialized toolkit submodule, executor не читает чужой authoritative workspace вместо своей worktree;
- `task_finalize` не требует локального patch внутри task run для commit/push path;
- trailing-junk payload на входе `runtime_v2_apply_event.sh` превращается в canonical JSON без `payloadWarning`;
- shell syntax и `runtime-v2` tests зелёные.
