# PL-113 — Micro-profile M1+M2 execution path

Цель:
- ввести отдельный `micro` execution profile для задач с маленькой целевой дельтой;
- убрать orchestration chatter из LLM-path и перенести finalize/metadata/diff сборку в deterministic scripts;
- заложить telemetry + budget envelope для контроля token cost без раннего включения жёсткого M3 enforcement.

Планируемый scope:
- добавить strict `micro_task_classifier` c deny-list для toolkit / infra / CI / auth / finalize / runtime-control задач;
- сделать `context_builder` единственным source of truth для file-context в micro-profile;
- собрать deterministic `canonical_diff`, `metadata_builder`, `micro_finalize` и `llm_call_telemetry`;
- прошить `executor_start`, `executor_build_prompt`, `executor_run` и `run.sh` под split `standard | micro`;
- добавить budget envelope (`max_calls`, `threshold_tokens`) и soft guard/reporting без полного hard-stop enforcement.

Definition of done:
- `executor_start` пишет `execution_profile.json`, а для `micro` инициализирует `execution_budget.json`;
- `executor_build_prompt` для `micro` использует только `context_builder`, а не generic long prompt;
- `executor_run` для `micro` идёт через path `solve -> deterministic checks -> optional repair -> deterministic finalize`;
- per-call telemetry пишет `llm_calls.jsonl` и агрегирует token budget;
- новые commands доступны через `.flow/shared/scripts/run.sh`;
- shell syntax и `runtime-v2` tests зелёные.
