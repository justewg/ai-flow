# PL-115 — Local micro canary: repair path

Цель:
- подтвердить, что `micro` profile корректно использует второй, и только второй, LLM call для repair-path;
- убедиться, что `micro_repair_prompt` собирается только из `canonical_diff` + failed checks;
- проверить, что общий token budget остаётся внутри целевого envelope.

Планируемый scope:
- расширить synthetic canary harness repair-mode сценария;
- заставить первый call дать неполную правку и провалить deterministic verification;
- подтвердить, что второй call закрывает failed checks и остаётся внутри `maxCalls=2`.

Definition of done:
- synthetic repair canary выполняет ровно два LLM calls (`implementation`, `repair`);
- second call uses deterministic repair prompt and не уходит в лишние rereads/discovery;
- итоговый `execution_budget.json` показывает `callCount=2`, `profileBreach=false`.
