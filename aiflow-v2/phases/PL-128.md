# PL-128 — Standardized Task Spec builder

Цель:
- превратить свободную постановку задачи в execution-ready normalized spec.

Планируемый scope:
- builder должен выдавать:
  - `profileDecision`
  - `decisionReason`
  - `interpretedIntent`
  - `candidateTargetFiles`
  - `expectedChange`
  - `checks`
  - `confidence`
  - `notes`
  - `rationale`
- сделать spec repository-aware через file existence и repo-root fallback.

Definition of done:
- normalized spec materialize-ится как отдельный JSON;
- spec пригоден как execution contract без чтения raw issue body;
- local smoke по replicated task body даёт корректные target files / checks / confidence.
