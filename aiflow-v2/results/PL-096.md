# PL-096 Result Log

Статус: `done`

## Цель фазы
- начать читать `runtime-v2` verdicts перед expensive/recovery paths legacy runtime;
- не делать full cutover, а держать legacy shell как transport/effect layer;
- подключить rollout/budget/stale gating через отдельный `v2` contour.

## Что реализовано
- Добавлен reusable bridge в `runtime-v2`:
  - `syncLegacyShadowSnapshot()`
  - `evaluateLegacyPolicyGate()`
- Legacy shadow sync теперь импортирует не только active/wait/review state, но и historical execution records из `execution_ledger.jsonl`.
- Для legacy execution history в `runtime-v2` materialize-ятся `Execution` и `legacy.execution_recorded` events, так что budget gate считает rough execution count уже не по пустому store.
- Добавлен новый `v2`-prefixed gate entrypoint:
  - `.flow/shared/runtime-v2/bin/runtime_v2_gate.js`
  - `.flow/shared/scripts/runtime_v2_gate.sh`
  - `.flow/shared/scripts/run.sh runtime_v2_gate`
- `executor_start.sh` теперь до запуска Codex читает `runtime-v2` gate verdict и не стартует expensive path при rollout/budget deny.
- `daemon_tick.sh` теперь читает `runtime-v2` gate:
  - перед новым claim;
  - перед resume из review feedback.
- `watchdog_tick.sh` теперь читает `runtime-v2` gate перед recovery action и уходит в `POLICY_HOLD`, если `runtime-v2` не разрешает recovery path.
- Все новые integration entrypoints оставлены в явном `v2`-контуре и не смешаны со старой ad-hoc логикой.
