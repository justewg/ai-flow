# PL-097 Result Log

Статус: `done`

## Цель фазы
- начать переводить selected lifecycle transitions из legacy shell runtime в `runtime-v2 applyEvent()`;
- перестать держать review/wait/execution lifecycle только в legacy state files;
- сделать это как best-effort sidecar, без полного orchestrator cutover.

## Что реализовано
- Добавлен reusable event bridge:
  - `applyLegacyEventBridge()`
  - `runtime_v2_apply_event.js`
  - `runtime_v2_apply_event.sh`
  - `.flow/shared/scripts/run.sh runtime_v2_apply_event`
- Legacy shell scripts начали best-effort публиковать selected события в `runtime-v2`:
  - `task_ask.sh` -> `human.wait_requested`
  - `daemon_check_replies.sh` -> `human.response_received`
  - `task_finalize.sh` -> `review.finalized`
  - `executor_start.sh` -> `execution.started`
  - `executor_run.sh` -> `execution.finished`
- Bridge перед `applyEvent()` сначала sync-ит legacy shadow snapshot, так что `runtime-v2` продолжает видеть текущий legacy context и не теряет rough execution history.
- Исправлен orchestrator gap:
  - `human.response_received` теперь очищает старый `canonicalWaitCommentId`, чтобы следующий `review.finalized` мог установить новый review anchor.
- Исправлен shell wrapper bug:
  - `runtime_v2_apply_event.sh` больше не искажает `payload-json` лишней `}` из-за неверного default expansion.
