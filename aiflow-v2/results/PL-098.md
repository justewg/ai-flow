# PL-098 Result Log

Статус: `done`

## Цель фазы
- начать использовать `runtime-v2` как primary source для selected lifecycle contexts;
- перестать держать `executing/reviewing` только в legacy state files;
- сохранить cutover узким и обратимым.

## Что реализовано
- В store/adapters добавлены list-операции:
  - `listTasks()`
  - `listTaskStates()`
- Добавлен derived primary context layer:
  - `derivePrimaryContexts()`
  - `runtime_v2_primary_context.js`
  - `runtime_v2_primary_context.sh`
- Добавлен reconcile-wrapper:
  - `runtime_v2_reconcile_primary_context.sh`
- `runtime_v2` теперь выводит selected primary contexts:
  - `executing` -> active task context
  - `reviewing` -> review context
- Reconcile-wrapper проецирует эти contexts обратно в legacy files:
  - `daemon_active_task.txt`
  - `daemon_active_issue_number.txt`
  - `project_task_id.txt`
  - `daemon_review_task_id.txt`
  - `daemon_review_issue_number.txt`
  - `daemon_review_pr_number.txt`
  - плюс очистка stale review context при отсутствии primary review state
- `daemon_tick.sh` и `watchdog_tick.sh` теперь вызывают `runtime_v2_reconcile_primary_context.sh` перед основной логикой и тем самым начинают считать `runtime-v2` primary source для selected contexts.
