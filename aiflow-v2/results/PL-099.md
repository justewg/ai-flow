## Что реализовано
- В `applyEvent()` waiting-state теперь хранит structured waiting metadata в `taskState.meta.waiting`:
  - `kind`
  - `waitCommentId`
  - `commentUrl`
  - `pendingPost`
  - `waitingSince`
- `human.response_received` очищает `taskState.meta.waiting` при выходе из `waiting_human`.
- `derivePrimaryContexts()` расширен waiting-slice:
  - `taskId`
  - `issueNumber`
  - `waitCommentId`
  - `kind`
  - `commentUrl`
  - `pendingPost`
  - `waitingSince`
- `runtime_v2_primary_context.sh` теперь публикует waiting counters и waiting fields.
- `runtime_v2_reconcile_primary_context.sh` теперь восстанавливает legacy waiting context:
  - `daemon_waiting_issue_number.txt`
  - `daemon_waiting_task_id.txt`
  - `daemon_waiting_question_comment_id.txt`
  - `daemon_waiting_kind.txt`
  - `daemon_waiting_pending_post.txt`
  - `daemon_waiting_since_utc.txt`
  - `daemon_waiting_comment_url.txt`
- `task_ask.sh` теперь шлёт полный `human.wait_requested` payload:
  - для успешного immediate comment post;
  - для pending-post / outbox path тоже.
- Pending-post path теперь тоже делает executor stop request и оставляет `runtime-v2` waiting state синхронным с legacy wait.
