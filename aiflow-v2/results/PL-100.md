# PL-100 Result Log

Статус: `done`

## Цель фазы
- убрать прямые legacy business writes для `review/wait` transitions;
- сделать `runtime-v2` primary source не только для ordinary wait, но и для review-feedback waiting;
- оставить legacy shell transport-layer, которая эмитит события и потребляет reconcile.

## Что реализовано
- В orchestrator добавлен `review.feedback_wait_requested`, который:
  - сохраняет review-feedback anchor,
  - оставляет task в `reviewing`,
  - обновляет `canonicalWaitCommentId`,
  - хранит review-feedback waiting metadata в `taskState.meta.reviewFeedback`.
- `review.finalized` теперь тоже сохраняет review-feedback waiting metadata:
  - `commentUrl`
  - `pendingPost`
  - `waitingSince`
- `human.response_received` теперь очищает и ordinary waiting metadata, и review-feedback waiting metadata.
- `derivePrimaryContexts()` расширен review-slice:
  - `waitCommentId`
  - `commentUrl`
  - `pendingPost`
  - `waitingSince`
- `runtime_v2_primary_context.sh` публикует review-feedback anchor fields.
- `runtime_v2_reconcile_primary_context.sh` теперь умеет:
  - восстанавливать ordinary `waiting_human` из v2;
  - восстанавливать `REVIEW_FEEDBACK` waiting из `reviewing` state, когда у review есть anchor metadata.
- `task_ask.sh` переведён на `emit event -> reconcile` для ordinary wait:
  - immediate post path;
  - pending-post path.
- `daemon_check_replies.sh` теперь:
  - при clarification path эмитит:
    - `human.wait_requested` для ordinary wait;
    - `review.feedback_wait_requested` для review feedback;
  - после `human.response_received` делает reconcile вместо ручного waiting-state source update.
- `task_finalize.sh` теперь не пишет review/wait business state напрямую:
  - review-feedback context создаётся через `review.finalized`;
  - legacy review/wait files материализуются через reconcile.
- `daemon_tick.sh` review-feedback resume больше не делает direct `daemon_active_*` assignment перед executor path и опирается на transport data из reply probe.
