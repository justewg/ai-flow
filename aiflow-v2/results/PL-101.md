# PL-101 Result Log

Статус: `done`

## Цель фазы
- переделать watchdog в настоящий supervisor;
- остановить auto-healing behaviour, которая сама плодит side effects;
- оставить у watchdog только stop/freeze/alert semantics.

## Что реализовано
- Из `watchdog_tick.sh` убраны recovery paths вида:
  - `daemon_tick`
  - `daemon_uninstall`
  - `daemon_install`
- Вместо этого введены supervisor actions:
  - `FREEZE_SAFE`
  - `FREEZE_EMERGENCY`
  - `STOP_EXECUTOR_AND_FREEZE`
  - `ALERT_BRANCH_SYNC`
- Добавлены supervisor helpers:
  - `run_supervisor_stop_executor()`
  - `run_supervisor_set_mode()`
- `run_supervisor_set_mode()` теперь:
  - переводит runtime в `SAFE` или `EMERGENCY_STOP` через `containment_mode.sh`;
  - пишет incident через `incident_append.sh`.
- Mapping anomaly -> action изменён так:
  - `DAEMON_NOT_INSTALLED` -> `FREEZE_EMERGENCY`
  - `DAEMON_NOT_LOADED` -> `FREEZE_EMERGENCY`
  - `DAEMON_LOG_STALE_WITH_LOCK` -> `FREEZE_EMERGENCY`
  - `EXECUTOR_PID_DEAD` -> `STOP_EXECUTOR_AND_FREEZE`
  - `EXECUTOR_HEARTBEAT_STALE` -> `STOP_EXECUTOR_AND_FREEZE`
  - `DAEMON_IDLE_WITH_ACTIVE_TASK` -> `FREEZE_SAFE`
  - `ACTIVE_TASK_WITHOUT_EXECUTOR_STATE` -> `FREEZE_SAFE`
  - `BRANCH_SYNC_REQUIRED` -> `ALERT_BRANCH_SYNC`
- Runtime-v2 gate больше не решает, можно ли watchdog останавливать flow: supervisor stop/freeze path теперь authoritative.
- Добавлен `WATCHDOG_IGNORE_DIRTY=1` как explicit harness override для локальных smoke без изменения default behaviour.
- State names обновлены под supervisor semantics:
  - `SUPERVISOR_ACTION_PENDING`
  - `SUPERVISOR_ACTION_APPLIED`
  - `SUPERVISOR_ACTION_FAILED`

## Изменённые файлы

## Проверки
- `bash -n .flow/shared/scripts/watchdog_tick.sh`
- Local supervisor smoke в clean sandbox:
  - watchdog запущен с `WATCHDOG_IGNORE_DIRTY=1`;
  - `daemon_status` в sandbox даёт `NOT_INSTALLED`;
  - результат:
    - `CONTROL_MODE=EMERGENCY_STOP`
    - `watchdog_state.txt=SUPERVISOR_ACTION_APPLIED`
    - `watchdog_last_action.txt=FREEZE_EMERGENCY`
    - `incident_ledger.jsonl` получил новые записи

## Ограничения PL-101
- Watchdog пока ещё shell-based и не читает structured anomaly model из `runtime-v2`;
- supervisor умеет freeze/stop/alert, но richer anomaly classification и operator surfaces ещё впереди;
- следующий шаг — перенести budget/quota/circuit-breaker decisions в fully authoritative `runtime-v2`.
