- Добавлен `control_policy.js` с `deriveGlobalControlMode()`:
  - если есть хотя бы один `emergency_stop`, global mode = `EMERGENCY_STOP`
  - если есть `paused_budget` без emergency, global mode = `SAFE`
  - иначе `AUTO`
- Добавлен CLI:
  - `.flow/shared/runtime-v2/bin/runtime_v2_control_mode.js`
- Добавлен shell wrapper:
  - `.flow/shared/scripts/runtime_v2_sync_control_mode.sh`
- `runtime_v2_gate.sh` теперь после gate evaluation синхронизирует containment mode из `runtime-v2`.
- `executor_run.sh` quota path теперь:
  - эмитит `budget.breached`
  - вызывает `runtime_v2_sync_control_mode.sh`
  - не опирается как primary path на прямой `containment_mode set ...` вне `runtime-v2`
