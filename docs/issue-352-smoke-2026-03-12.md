# ISSUE-352 smoke report

Дата: 2026-03-12

## Что проверялось

- project-local `.flow` layout после перехода на host-level shared runtime;
- launchd daemon/watchdog для профиля `planka`;
- namespaced state layout с compatibility symlink layer;
- повторный executor-run для `ISSUE-352` после live-fix.

## Что подтвердилось

- канонический project config использует `.flow/config/flow.env`;
- runtime logs, state и launchd artifacts живут в host-level `.ai-flow` layout;
- daemon и watchdog снова держат реальные owner pid в namespaced lock directories;
- `status_snapshot` читает актуальный root-state вместо старого split-brain namespace;
- executor снова стартует автоматически из daemon-loop.

## Найденный live root cause

Новый state layout оставил `daemon.lock` и `watchdog.lock` как compatibility symlink-и, но shared toolkit продолжал делать `mkdir` по этим symlink paths. Из-за этого launchd-контур попадал в ложный `already running`, а daemon/watchdog не захватывали lock корректно.

Дополнительно executor в sandboxed `codex exec --full-auto` упирался в недоступный network path до backend и не мог надёжно пройти до `task_finalize`.

## Что исправлено вне git-дельты репозитория

- shared toolkit переведён на реальные namespaced lock paths для daemon/watchdog;
- `executor_run` получил переключатель `EXECUTOR_CODEX_BYPASS_SANDBOX`;
- `dev_commit_push` в shared toolkit перестал быть жёстко привязан к `development` и может пушить в рабочую head-ветку задачи.

## Что изменено в этом репозитории

- в `.flow/config/flow.sample.env` задокументирован флаг `EXECUTOR_CODEX_BYPASS_SANDBOX`;
- этот smoke-report добавлен как repo-side артефакт для `ISSUE-352`.

## Что осталось вне scope этого PR

- live fixes в host-level shared toolkit не входят в git-дерево PLANKA и должны оформляться отдельно как shared source of truth;
- локальные env файлы профиля не включаются в repo-side PR.
