# PL-033: финализация итеративного флоу после ответа пользователя

## Контекст
- Task ID: `PL-033`.
- Issue: `#45`.
- Пользовательский ответ в Issue: `Продолжай`.

## Проверенный сценарий
1. Агент выполнил первичный рабочий шаг (`commit1`) и обновил PR.
2. Агент опубликовал в Issue сигнал `CODEX_SIGNAL: AGENT_QUESTION` с явным вопросом `Продолжать?`.
3. Демон перешел в состояние ожидания ответа пользователя (`WAIT_USER_REPLY=1`).
4. После ответа пользователя демон зафиксировал `USER_REPLY_RECEIVED` и опубликовал `CODEX_SIGNAL: AGENT_RESUMED`.
5. Агент продолжил выполнение и довел задачу до финализации в `In Review`.

## Критерии результата
- Один Issue соответствует одному PR.
- В PR сохранены ровно 2 коммита по `PL-033`.
- В Issue присутствует обязательная пара сигналов: `AGENT_QUESTION` и `AGENT_RESUMED`.
- Финализация выполняется с финальным PR-сигналом `FINAL_REVIEW`.
