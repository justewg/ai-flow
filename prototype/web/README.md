# PL-003 Prototype (Web)

Минимальный прототип экрана 50/50 для задачи `PL-003`.

## Что реализовано
- Верхняя зона: крупный вывод текста.
- Нижняя зона: детерминированная клавиатура RU/EN.
- Явное переключение языка (кнопка).
- Переключение темы light/dark (кнопка с луной/солнцем справа вверху).
- Стабильный split-layout 50/50.
- Попытка фиксации landscape через `screen.orientation.lock("landscape")`.
- Явный fallback в portrait без текста: крупная мнемоника поворота (без кнопок).
- Сохранение/восстановление текста и языка в `localStorage`.
- Без автокоррекции.
- Без скрытых состояний.
- Интерфейс родительского контроля:
  - Триггер входа по механике из обсуждения: удержание `Р` + обе громкости.
  - Для desktop-демо аппаратные кнопки эмулируются через `F9` + `F10`, а также fallback `+` + `-` (и `Alt+↑` + `Alt+↓`).
  - Режим триггера выбирается автоматически:
    - `prod` (по умолчанию не на iOS/iPadOS): реальные `AudioVolumeUp/AudioVolumeDown` + fallback desktop-клавиш.
    - `dev` (по умолчанию на iPhone/iPad): экранные `VOL+/VOL-` (можно нажимать по очереди в окне триггера).
  - Важно: в мобильном Safari аппаратные кнопки громкости iPhone/iPad не отдаются в JS-события, поэтому для web-прототипа используется экранная эмуляция `VOL+/VOL-`.
  - Режим можно переопределить query-параметром: `?parentTriggerMode=prod` или `?parentTriggerMode=dev`.
  - Экран ввода PIN (уровень 1) и родительская панель быстрых функций.
  - Отдельный PIN уровня 2 для перехода в системные настройки.

## Быстрый запуск
1. Открой `prototype/web/index.html` в браузере.
2. Переключи окно в landscape для проверки основного сценария.
3. Для входа в родительский режим:
   - `prod`: удерживай `Р` и нажми обе громкости (или `F9`+`F10`, или `+`+`-`), затем введи PIN.
   - `dev` (iPhone/iPad): удерживай `Р` и нажми `VOL+` и `VOL-` на экране (можно по очереди), затем введи PIN.

## Ограничения текущей версии
- Это UI-прототип, не kiosk-режим.
- Нет TTS.
- Нет датчиков наклона.
- Нет реального выхода в Android из web-прототипа (только интерфейс и flow).
- Поведение clear/backspace hold будет дорабатываться в задачах `PL-005+`.
